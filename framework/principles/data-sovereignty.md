# Data Sovereignty

*Status: standing principle*

---

Data sovereignty is a fundamental human right. No one should be
locked into a platform because their context, memory, and identity
are trapped there.

## What this means in practice

- User data is stored in formats the user controls (markdown files
  in a repo they own)
- No proprietary formats that require specific tools to read
- Migration between platforms must always be possible
- Local-first architecture wherever feasible
- Privacy-first: sensitive data stays in private repos, never in
  public git history
- The user can disconnect from any AI platform and retain all
  their knowledge, context, and operating rules

## How this applies to product decisions

- Open-source tooling and protocols are the default
- Paid products add convenience and experience, not lock-in
- If a feature would make migration harder, it needs explicit
  justification
- International compliance complexity is bypassed through
  local-first architecture (the user holds their own data)

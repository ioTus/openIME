# Principles

Standing decisions and values that apply across all your work.
These are treated as settled unless you explicitly reopen them.

## Framework Principles (ship with IME)

These define how IME itself operates:

- **Data Sovereignty** — your data lives in files you control
- **PARA Knowledge Organization** — how the knowledge layer
  is structured
- **Connection Failure Protocol** — how to handle bridge outages

rules that you want to apply across all projects and all
tools. Examples:

- Product design philosophy
- Communication preferences
- Business ethics and boundaries
- Personal operating rules

Create a new `.md` file in this directory for each principle.
